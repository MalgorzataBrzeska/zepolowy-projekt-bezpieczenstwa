# FIX-001 - Naprawa błędu FIRE-4512
## Opis
Został skonfigurowany firewall w trybie deny incoming oraz włączone zostały tylko i wyłącznie wymagane porty.
 System wykrywał załadowany model iptables aczkolwiek nie było aktywnych reguł zabezpieczeń.

## Dokonane zmiany
1. Ustawiono domyślną politykę: deny incoming, allow outgoing.
2. Włączono firewall.
3. Otworzono port SSH i porty dla usług HTTPS/HTTP (jeżeli wymagane).

## Opis aktualnego stanu
Po wprowadzeniu poprawek systemowych został przeprowadzony test, który wykazuje że porty nieużywane pokazują status closed/filtered, dostęp do systemu z nieautoryzowanych sieci jest zabroniony oraz reguły są aktywne i potwierdzone poleceniem ufw status verbose

## Wynik i status
SUKCES - firewall działa wraz z aktywnymi regułami
